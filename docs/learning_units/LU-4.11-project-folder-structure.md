# Learning Unit 4.11: Creating a Project Folder Structure for Data Science Work

## Objective

Design a clear, maintainable folder structure for FraudScope so all future units can plug into the same workflow.

## Product-Oriented Implementation

1. Define standard directories for data stages, notebooks, outputs, and source modules.
2. Add an automated structure checker to validate required folders.
3. Add notebook guidance so contributors can follow the same structure.

## Deliverables in this unit

- `src/fraudscope/folder_structure.py`
- `notebooks/LU-4.11-project-folder-structure.ipynb`
- LU implementation note and README update

## Why this matters for fraud detection

Fraud analytics projects quickly become messy. Consistent folder structure keeps experiments reproducible and simplifies audit trails.
