# Learning Unit 4.5: Installing Python and Anaconda on the Local Machine

## Objective

Set up the local machine with Python + Anaconda and understand repository intent so future units can build a real fraud detection product.

## Product-Oriented Implementation

1. Confirm a repository layout designed for the complete anomaly detection lifecycle.
2. Define where raw data, processed data, analysis notebooks, and outputs will live.
3. Create a reusable source package for future data loading and anomaly modules.

## Deliverables in this unit

- DS-first folder structure for FraudScope.
- Starter Python package namespace (`src/fraudscope`).
- Baseline project README with LU continuity context.

## Why this matters for fraud detection

Anomaly detection projects fail early when structure is unclear. This unit creates a consistent workspace to keep ingestion, cleaning, scoring, and reporting reproducible.
