# Learning Unit 4.8: Understanding Notebook Cells - Code vs Markdown

## Objective

Distinguish code cells from markdown cells and apply both intentionally in FraudScope notebooks.

## Product-Oriented Implementation

1. Define what belongs in code cells (execution) versus markdown cells (explanation).
2. Build a notebook example that combines narrative and runnable logic.
3. Establish notebook communication standards for anomaly analysis reports.

## Deliverables in this unit

- `src/fraudscope/notebook_cell_guide.py`
- `notebooks/LU-4.8-code-vs-markdown-cells.ipynb`
- LU implementation note and README update

## Why this matters for fraud detection

Fraud investigations need both evidence and explanation. Code cells generate evidence, markdown cells communicate reasoning to reviewers.
