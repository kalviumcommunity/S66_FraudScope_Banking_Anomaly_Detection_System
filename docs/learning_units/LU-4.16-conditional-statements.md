# Learning Unit 4.16: Writing Conditional Statements for Data Logic

## Objective

Apply `if`, `elif`, and `else` logic to classify transaction risk in a rule-driven way.

## Product-Oriented Implementation

1. Define threshold-based conditional labels for suspicious behavior.
2. Add a reusable function that converts transaction amount to risk band.
3. Demonstrate conditions in notebook examples for analyst interpretation.

## Deliverables in this unit

- `src/fraudscope/conditional_rules.py`
- `notebooks/LU-4.16-conditional-logic.ipynb`
- LU implementation note and README update

## Why this matters for fraud detection

Early fraud systems depend on simple and explainable rules. Conditions are the first step before statistical scoring.
