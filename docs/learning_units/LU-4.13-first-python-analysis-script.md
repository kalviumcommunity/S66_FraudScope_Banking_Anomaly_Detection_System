# Learning Unit 4.13: Creating and Running a First Python Script for Data Analysis

## Objective

Write and execute a basic Python script that reads transaction data and outputs useful summary insights.

## Product-Oriented Implementation

1. Add a small sample transaction dataset in `data/raw`.
2. Add a script that loads data and computes basic summary metrics.
3. Save summary output in `outputs/reports` for traceable analysis.

## Deliverables in this unit

- `data/raw/sample_transactions.csv`
- `scripts/lu_4_13_first_analysis.py`
- `notebooks/LU-4.13-first-python-analysis-script.ipynb`
- `outputs/reports/lu_4_13_summary.txt` (generated)
- LU implementation note and README update

## Why this matters for fraud detection

This is the first end-to-end executable step: data in, analysis logic, summary out. It forms the base for future anomaly scoring scripts.
