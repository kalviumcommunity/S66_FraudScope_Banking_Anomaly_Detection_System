# Learning Unit 4.6: Verifying Python, Conda, and Jupyter Installation

## Objective

Validate local Python tooling and establish a notebook-first workflow that supports reproducible fraud analysis.

## Product-Oriented Implementation

1. Add an automated environment verification script for `python`, `conda`, and `jupyter` commands.
2. Add a notebook that demonstrates both setup checks and core Python syntax patterns used in data workflows.
3. Keep repository continuity by documenting how this unit enables future ETL, EDA, and anomaly scoring units.

## Deliverables in this unit

- Environment verification module in `src/fraudscope/environment_verify.py`
- Hands-on notebook in `notebooks/LU-4.6-environment-verification.ipynb`
- LU implementation note and README update

## Why this matters for fraud detection

Without a stable local toolchain, transaction data preparation and anomaly experiments become unreliable across machines. This unit ensures every next Learning Unit runs consistently.
